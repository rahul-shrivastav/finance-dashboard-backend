Finance Dashboard API

Complete API documentation

Auth

POST api/auth/signup
Creates a new user account with role-based access.
{ "id": "user_3", "name": "New User", "email": "user@test.com", "password": "123456", "roleId": 3, "status": "active" }
POST api/auth/login
Authenticates user and returns a JWT token.
{ "email": "user@test.com", "password": "123456" }

Users (Admin Only)

GET api/users?page=1&limit=10
Fetch all users with pagination support.
GET api/users/:id
Fetch details of a specific user by ID.
POST api/users
Create a new user (admin only).
{ "id": "user_5", "name": "Test User", "email": "test@test.com", "password": "123456", "roleId": 2, "status": "active" }
PUT api/users/:id
Update user details such as role or status.
{ "name": "Updated Name", "email": "updated@test.com", "roleId": 2, "status": "inactive" }
DELETE api/users/:id
Delete a user from the system.

Transactions

GET api/transactions
Fetch transactions with filtering and pagination.
api/transactions?page=1&limit=5&type=expense&userId=user_1
POST api/transactions
Create a new transaction (admin only).
{ "userId": "user_1", "categoryId": 1, "type": "expense", "amount": 500, "note": "Lunch" }
PUT api/transactions/:id
Update an existing transaction.
{ "type": "income", "amount": 1000, "categoryId": 2, "note": "Updated" }
DELETE api/transactions/:id
Delete a transaction.

Trends

GET api/aggregate/insights
Returns weekly, monthly, and yearly financial summaries.
GET api/aggregate/insights/user/:userId
Fetch trends for a specific user.
GET api/aggregate/categories
Returns category-wise transaction counts over time.

Notes

- All protected routes require Authorization header
- Format: Bearer TOKEN
- Rate limiting applied on auth routes
- SQLite database resets on restart (seeded automatically)
